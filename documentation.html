Buy Ticket Endpoint

This endpoint allows a user to purchase a ticket for a specific product. An optional campaign code can be provided.

Endpoint

POST /api/buy-ticket

Headers

Content-Type: application/json

Request Body

The request body must be valid JSON with the following fields:

Field Type Required Description
product_id integer Yes ID of the product/ticket being purchased
user_id integer Yes ID of the user making the purchase
campaign_code string No Optional promotional or campaign code

Example Request

{
    "product_id": 42,
    "user_id": 7,
    "campaign_code": "SUMMER2026"
}

Responses

Success

On success, the ticket purchase is processed via the internal buyTicket() method.

(Exact success response depends on the APIHandler implementation.)

Error Response

Returned when invalid data types are provided.

{
    "status": "error",
    "message": "Invalid data types"
}

Notes

  • The request body is read from php://input
  • All responses are returned as JSON
  • The endpoint assumes authentication/authorization is handled elsewhere
